# Исправление ошибки UUID parsing для /plans/generate

## Проблема

При попытке вызвать endpoint `/api/content/plans/generate` возникает ошибка:
```json
{
  "detail": [{
    "type": "uuid_parsing",
    "loc": ["path", "plan_id"],
    "msg": "Input should be a valid UUID, invalid character: expected an optional prefix of `urn:uuid:` followed by [0-9a-fA-F-], found `g` at 1",
    "input": "generate"
  }]
}
```

## Причина

FastAPI пытается интерпретировать "generate" как UUID в параметре `plan_id` роута `/plans/{plan_id}`. Это происходит из-за неправильного порядка определения роутов.

## Решение

Порядок роутов в FastAPI имеет критическое значение. Специфичные роуты (например, `/plans/generate`) должны быть определены **ПЕРЕД** параметризованными роутами (например, `/plans/{plan_id}`).

### Правильный порядок роутов:

1. ✅ `/plans/generate` (POST) - специфичный роут
2. ✅ `/plans` (GET) - список планов
3. ✅ `/plans/{plan_id}` (GET) - параметризованный роут

### Что было исправлено:

1. Перемещен роут `/plans/generate` в начало списка роутов с `/plans`
2. Добавлен комментарий о важности порядка роутов
3. Убрано дублирование кода логирования

## Проверка

После исправления:

1. Перезапустите backend
2. Попробуйте снова вызвать `/api/content/plans/generate`
3. Ошибка должна исчезнуть

## Дополнительная информация

FastAPI проверяет роуты в порядке их определения. Если параметризованный роут определен раньше специфичного, FastAPI может попытаться сопоставить путь с параметризованным роутом, что приводит к ошибке валидации типа.
